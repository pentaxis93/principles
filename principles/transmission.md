# Transmission

**Movement:** IV — Land · [Index](../README.md)

> **Invariant:** Work completes when the recipient can act on it, not
> when the maker finishes creating it.

## The universal

Every artifact has a recipient: an operator, an executor, a reviewer, a
future instance of the maker crossing a context boundary, an external
contributor landing cold. The artifact must carry what that recipient
needs, in the register they can use, without forcing them to
reconstruct the maker's context or journey. The door must fit who
enters: compression is limited by the recipient's capacity to cross
into understanding, and artifacts only the makers can parse are walls,
not doors.

Transmission is not more explanation; it is fit between artifact and
recipient. The gratitude test applies: would the recipient thank you
for how you structured this? If not, transmission failed regardless of
how correct the content is.

**Recognition:** The artifact preserves the maker's journey, assumes
private context, mixes registers, buries the actionable claim, or would
force the recipient to ask what the maker already knew.

**Corrective:** Name the recipient and the action they must take. Remove
material that does not change that action; add only what the recipient
needs to proceed.

## Projections

### Interfaces describe themselves *(machine interfaces)*

An interface that exposes its own schema lets consumers — humans landing
cold, autonomous agents, code generators — discover capability through
the interface itself, without operator-mediated discovery. Every fresh
context is the first time a consumer encounters a surface; the surface
that describes itself is the door that fits them. The schema is the
contract, not a paraphrase of the contract.
— *Source: commons `DESIGN-PRINCIPLES.md` §17. Edge to
[Single Home](single-home.md): documentation derived from schema cannot
drift; maintained alongside, it is a second source of truth.*

### Uniform patterns across surfaces *(ecosystems)*

Pagination shape, error structure, identifier format — these are
cognitive load that amortizes if uniform across an ecosystem and
compounds if inconsistent. When two surfaces disagree, every consumer
pays the integration cost twice, and the inconsistency teaches new
surfaces that disagreement is allowed.
— *Source: commons `DESIGN-PRINCIPLES.md` §19.*
